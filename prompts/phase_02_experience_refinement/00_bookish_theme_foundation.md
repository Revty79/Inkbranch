# 00 — Bookish Theme Foundation

## Objective

Refresh Inkbranch’s visual language so the app feels warm, literary, immersive, and comfortable.

The goal is to make Inkbranch feel less like a prototype dashboard and more like a story-first reading product.

---

## Product Context

Phase 01 established the functional platform.

Phase 02 begins by strengthening identity.

Inkbranch should now feel like:
- a living book
- a reader’s space
- a warm study or library
- paper, ink, cloth, leather, wood, candlelight
- immersive and calm
- elegant without becoming noisy or ornate

This prompt is the visual and shell foundation for the rest of the phase.

---

## Scope of This Prompt

Implement the design-system and shell-level changes needed to establish the Phase 02 product theme.

This includes:

- color/token refinement
- surface styling refinement
- typography rhythm refinement
- shell polish
- card and form styling improvements
- general layout comfort improvements
- small atmospheric details where appropriate

This prompt does **not** require large reader-flow or creator-flow restructuring yet.
That comes in later prompts.

---

## Theme Direction

### Desired feel
- warm parchment
- deep ink text
- soft sepia / amber / muted brown accents
- comfortable page surfaces
- restrained shadows
- quiet elegance
- intimate reading space

### Avoid
- neon colors
- sterile white productivity panels
- high-gloss modern SaaS aesthetics
- glassmorphism
- visual clutter
- overly decorative fantasy UI

---

## Typography Direction

Inkbranch should now feel more intentionally literary.

Preserve readability first.

The UI should distinguish between:
- story prose
- section labels
- navigation
- creator form UI
- Chronicle metadata

Improve:
- rhythm
- vertical spacing
- measure width
- paragraph comfort
- heading hierarchy

The app should reward long-form reading.

---

## Surface and Layout Direction

Reader-facing surfaces should feel more like:
- pages
- folios
- quiet reading panels

Admin-facing surfaces should still fit the same brand, but they may remain more structured and utilitarian.

The shell should feel:
- calmer
- cleaner
- more intentional
- more like a branded product

---

## Required Areas To Touch

At minimum, improve:

- global theme variables/tokens
- root layout feel
- public landing presence
- signed-in shell/navigation
- general card styling
- general button styling
- general form field styling
- spacing and content width discipline

If the app already has theme tokens, refine them rather than replacing them carelessly.

---

## Design Rules

### Reader-first comfort
Long-form reading comfort is more important than flashy styling.

### Consistency
The same visual language should carry across:
- public site
- app shell
- reader screens
- creator/admin sections

### Restraint
A little atmosphere goes a long way.
Do not bury usability under theme flourish.

---

## Deliverables

By the end of this prompt, the project should include:

- refined theme tokens or global styling
- stronger shell aesthetics
- stronger component baseline styling
- a clearly more bookish and comfortable visual direction
- a reusable styling base for later prompts

---

## Acceptance Criteria

This prompt is complete when:

1. the app no longer feels like a generic early scaffold
2. the shell and common components reflect a warm literary identity
3. typography and spacing are more comfortable for reading
4. the design system is ready for the reader overhaul in the next prompt
5. the product feels more like Inkbranch

---

## Verification

At minimum, verify:

- landing page feel
- signed-in shell feel
- cards/forms/buttons consistency
- mobile readability
- no major regressions to layout or accessibility

---

## Constraints

Do not:
- spend this prompt rebuilding large product flows
- overdecorate at the expense of clarity
- create a theme that feels gimmicky or themed-for-theme’s-sake

This prompt should make the whole app feel better before deeper UX work begins.
